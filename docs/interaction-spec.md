# mnote 交互规格（主线）

本文档定义当前阶段唯一主线交互，目标是让实现可用、可测、可迭代。  
范围与架构约束以 [`architecture.md`](architecture.md) 为准，视觉风格以 [`ui-design.md`](ui-design.md) 为准。

---

## 1. 信息架构

- `root`：唯一根目录，用户手动选择，不自动猜测。
- `notebook`：`root` 下一层子文件夹，文件夹名即笔记本名。
- `workspace`：当前选中的 notebook 目录。

---

## 2. 页面布局（当前）

- 左栏：root 卡片 + 新建笔记本输入区 + 笔记本列表。
- 中栏：当前 workspace 概览（后续扩展为文件树）。
- 右栏：编辑区占位（后续接入编辑/预览套壳）。

---

## 3. 核心交互流程

### 3.1 启动

1. 启动读取 `~/.mnote/config.json`。
2. 若有 root 配置且可恢复：加载 notebooks。
3. 若无 root：自动弹出目录选择面板。

### 3.2 选择根目录

1. 用户点击「选择根目录 / 更换根目录」按钮。
2. 选择目录后立即保存 root 书签，并清空上次 workspace 记录。
3. 刷新 notebook 列表并应用默认选择规则：
   - 优先恢复最近 workspace；
   - 否则选第一个 notebook（按名字排序）；
   - 若没有 notebook，停留空状态。

### 3.3 新建笔记本（输入名称即文件夹名）

1. 用户在输入框输入名称，按回车或点击「创建」。
2. 程序在 `root/<name>` 创建文件夹。
3. 成功后：
   - 刷新列表；
   - 自动选中新建 notebook；
   - 清空输入框并保持输入焦点。
4. 失败后弹提示，输入内容保留。

### 3.4 切换笔记本

1. 用户点击左栏列表项。
2. 程序将其设为当前 workspace，并写入配置。
3. 中栏与后续文件浏览均基于该 workspace。

---

## 4. 输入框行为约定（重点）

- 输入框必须始终可编辑（root 已设置时）。
- 创建按钮在输入为空白时禁用。
- 输入字符串会做 `trim`（去首尾空白）后再参与创建。
- 以下情况弹提示并阻止创建：
  - 未选择 root；
  - 名称为空；
  - 名称包含路径分隔符（至少 `/`）；
  - 同名普通文件已存在。
- 同名目录已存在时，不报错，直接选中该 notebook。

---

## 5. 错误提示文案规则

- 提示优先说明“如何修复”，避免只显示技术错误。
- 技术错误（例如系统 I/O 失败）可附加系统 `localizedDescription`。
- 统一使用弹窗提示，避免静默失败。

---

## 6. 验收清单（手工）

1. 首次启动自动弹 root 选择。
2. 选 root 后可在输入框正常输入文本。
3. 回车创建可成功生成 `root/<name>`。
4. 创建成功后列表出现条目并自动选中。
5. 重启后能恢复上次选中的 notebook。
6. 输入非法名称有明确报错，且输入框内容不丢失。
