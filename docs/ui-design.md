# mnote：UI 设计方向

本文档描述 **界面与视觉语言**（液态玻璃）。**技术架构与套壳策略**见 [`architecture.md`](architecture.md)。

---

## 1. 总风格：液态玻璃（Liquid Glass）

**结论：整体视觉采用液态玻璃风格**——强调材质层次、通透与纵深，与 macOS 原生的 **材质（Material）与振动（Vibrancy）** 语言一致，避免扁平硬切。

---

## 2. 设计意图（非像素规范）

实现时以系统 API 与设计 token 为准，以下为方向性约定：

| 方向 | 说明 |
|------|------|
| 背景与容器 | 使用 **模糊材质**（如 SwiftUI 的 `.ultraThinMaterial` / `.thinMaterial` 等）或 AppKit `NSVisualEffectView` 作为侧栏、顶栏、浮层面板背景 |
| 层次 | 侧栏 / 主编辑区 / 预览区通过 **材质强度或圆角** 区分层级，形成「玻璃叠在玻璃上」的液态感 |
| 边框与高光 | **细描边 + 柔和高光**（低对比），避免厚重实色框 |
| 圆角 | 面板与关键卡片采用 **统一圆角体系**（具体数值在 UI 稿或设计 token 中定义） |
| 动效 | 展开/切换 workspace 时使用 **轻量过渡**（opacity、blur 或 layout），流畅且不抢内容 |

---

## 3. 实现注意

- **可读性优先**：正文编辑区背景不宜过度依赖强模糊；必要时编辑区偏实色，侧栏与 chrome 使用玻璃材质。
- **深色 / 浅色**：两套主题下对比度需单独检查，尤其是玻璃上的次要文字。
- **嵌入原生视图**：`NSTextView`、`WKWebView` 等与 SwiftUI 材质并存时，注意 **边距与圆角裁剪**，避免「实色板压在玻璃上」的割裂感。
